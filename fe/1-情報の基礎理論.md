## 1-03 補数表現と固定小数点表示

**補数**

* N進数には[N-1の補数]と[Nの補数]がある
  * 10進数には9の補数と10の補数がある
* 「ある数」に[N-1の補数]を補うと、与えられた桁数の最大値になり、[Nの補数]を補うと、次のけたに桁上がりする
  * 10進数123に9の補数876を補うと999（3桁の最大）、10の補数877を補うと1000（桁上がり）になる
* 補数を使って負の数を表現する
* 2進数は1の補数と2の補数があり、2進数4ビットで「１の補数ならビット反転」、2の補数なら、「１の補数に１を加える」
 * 0101の1の補数は1010、0101の2の補数は1011

**固定小数点**

* 固定小数点表示（2の補数表現）
* nビットで表現できる整数の範囲 → -2^(n-1) ~ -2^(n-1)-1

## 1-05 誤差

* 誤差の種類
  * けたあふれ誤差 → 表現できる範囲を超えた（オーバーフロー、アンダーフロー）
  * 丸め誤差 → 指定された有効桁数で表現するための切り捨て切り上げ四捨五入などによる誤差
  * 桁落ち → 絶対値がほぼ等しい二つの数の絶対値の差を求めたときに、有効桁数が減るために発生する誤差
  * 情報落ち → 絶対値の非常に大きな数と小さな数の加減を行ったときに、小さい数が演算結果に反映されないために発生する誤差
  * 打ち切り誤差 → 浮動小数点数の計算処理の打ち切りを、指定した規則で行うことによって発生

## 1-06 シフト演算

n進数において、桁をずらすとnで割ったことになる。  
10進数123なら、1230なら10かけ、12.3なら10で割る、  
2進数10なら、100で2かけ、1なら2で割ったことになる。

シフト演算には、「符号を考慮しないシフト演算」と「符号を考慮するシフト演算」がある。

* 論理シフト  
論理シフトは符号を考慮しない。あふれたビットは捨てられ、空いたビットには0がはいる。

* 算術シフト
算術シフトは符号を考慮する。  
左シフトは、符号ビットはそのままに、溢れたビットを捨て、空いたビットに0をいれる。
右シフトは、符号ビットはそのまま、溢れたビットを捨て、あいたビットには**符号と同じビット**がはいる。

## 1-08 文字コード

文字コードとは、文字の一つ一つに特定の2進数を割り与えたもの。
JIS,ASCII、EBCDICなど。

* JIS
Japan Industrial Standard

* シフトJIS
1バイト文字（英数字・カナ）と2バイト文字（ひらがな・漢字）の混在が可能

* ASCII
American Standard Code for Infomation Intercahnge  
7ビットで表現し、8ビット目はパリティ  
英字、数字、特殊文字及び制御文字、米国標準なので漢字はない  

* EBCDIC
Extended Binary Coded Decimal Interchange Code  
IBMが開発、8ビット、汎用機を中心に普及

* EUC
Extended Unix Code
全角文字と全角カタナカを２バイトまたは３バイトで表現、Unixでの多言語対応
UNIXで漢字が使える

* UNICODE
世界各国の文字の多くを一つの体系で表現したもの
単一（同一）のコードで、各国の文字を統一

1と0の二つを扱う演算が論理演算

### 否定論理和回路（NOR回路）

## 1-10 半加算器と全加算器

論理回路の組み合わせで加算を表現する
2進数の加算を行う回路を加算器という。加算器には**半加算器**と**全加算器**がある。  

* 半加算器 
上位桁への繰り上がりを考慮する

* 全加算器
下位けたからの桁上がりと、上位桁への桁上がりを考慮する
