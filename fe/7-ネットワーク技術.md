# ネットワーク技術

## 7-01 電気通信サービス

**WAN(Wide Area Network)**  
遠隔地にあるLAN同士や、端末と汎用コンピュータを接続する広域ネットワーク

* 専用線サービス  
  特定の相手先との間を固定的に接続して行う通信サービス  

* 回線交換サービス  
  通信のたびにダイヤルすることによって相手先を呼び出し接続して行う通信サービス

* パケット交換サービス  
  データを一定の大きさ（パケット）に分割し、宛先情報などを付加して行うデータ通信サービス  

* フレームリレーサービス  
  高品質なデジタルネットワークを使用すること前提に、ネットワーク内で伝送誤りを訂正しつつ、高速通信を実現するサービス

* ATM(Asynchronus sTransfer Mode)  
  ATMは、あらゆる種類のデータを、セルと呼ばれる固定長（53バイト）のブロックスに分割して伝送する。セルリレーともいう

* ADSL(Asymmetric Digital Subscriber Line)  
  ADSLは既存の電話回線を利用して行う通信サービス
  上りよりも下りのほうが早い

* CATV(Community Antenna TeleVision)
  ケーブルテレビの回線を利用して行う接続サービス  
  ケーブルモデムが必要

* FTTH(Fiber To The Home)  
  各家庭まで光ファイバを敷設電話やインターネット、テレビなどを統合した高速データ通信を行うサービス

**モバイル通信**  
スマートフォンやタブレット通信するときは、通信事業者との契約で提供されるSIMカードを端末に挿入して行う。  
LTE, テザリング, MVNOなど。

## 7-02 OSI基本参照モデルとTCP/IP

**OSI基本参照モデル（Open Systemcs Interconnection)**  
ネットワークアーキテクチャの標準化

∥ 層 | 名称 | TCP/IP |
| - | - | - |
| 7層 | アプリケーション層 | アプリケーション層 |
| 6層 | プレゼンテーション層 | アプリケーション層 |
| 5層 | セッション層 | アプリケーション層 |
| 4層 | トランスポート層 | トランスポート層 |
| 3層 | ネットワーク層 | インターネット層 |
| 2層 | データリンク層 | ネットワークインタフェイス層 |
| 1層 | 物理層 | ネットワークインタフェイス層 |

上からアプセトネデブと覚えさせられるらしい  

**TCP/IP**  
OSI基本参照モデルにおいてネットワークアーキテクチャが標準化されているが、  
実際にはTCP/IPがRFCで基底され、インターネットで広く使われているプロトコル

| プロトコル名 | 概要 |
| - | - |
| HTTP | Hyptertext Transfer Protocol. WWWサーバとWWWクライアント間のHTMLをサポート |
| FTP | File Transfer Protocol. ファイルの送受信に利用する |
| TELNET | ホストにリモートログインするのに使う |
| SNMP | Simple Network Management Protocol. 構成機器管理やネットワーク機器の監視に使う |
| NTP | Network Time Protocol. タイムサーバと通信し、時刻調整を行う |
| SMTP | メールの送信に使う |
| POP | メールの受信に使う |
| IMAP | メールサーバのメールボックスからメールを取り出すときに使う |
| MIME | 電子メールで送受信できるデータとして、テキストだけでなく、画像や音声などもサポートする |

## 7-03 IPアドレス

TCP/IP のネットワークでは、IPアドレスと呼ばれる識別番号が端末やネットワーク機器に割り与えらえる。    
現在広く使われているIPv4では、IPアドレスを2進数32ビットで表現し、ネットワーク番号とホスト番号で構成する。    
2進数のままでは見難いので、それを10進数に変換して8ビット単位でドットで区切って表現する。  

**DNS(Domain Name System)**  
ドメイン名とIPアドレスを紐づけ、変換する。  

**グローバルアドレスとプライベートアドレス**  

* グローバルアドレス  
  インターネットに接続するための、世界で一意のIPアドレス
  NIC(Network Infomation Center)が発行する

* プライベートIPアドレス  
  LANやイントラネットなどの独立した網内で利用する

**DHCP(Dynamic Host Configuration Protocol)**  
動的にIPアドレスを割り当てる

**ポート番号**  
TCP/IPネットワークでは、IPアドレスによって通史泣いてのホストを特定する。  
さらに、アプリケーションを識別する番号がポート番号  
65535まで使え、0-1023まではwellKnownとして予約されている  

**NATとNAPT**

* NAT(Network Address Translation)  
  一つのグローバルIPアドレスに一つのプライベートアドレスを対応させる。  

* NAPT(Network Address Port Translation)  
  一つのグローバルアドレスに複数のプライベートアドレスを対応させる。

## 7-04 クラスとサブネット分割

IPアドレスは「ネットワーク部」「ホスト部」で構成される  
32ビットのうちどこまでをネットワーク部として、どこからをホスト部とするか、によって  
クラスA,クラスB,クラスC,クラスD に分類される  

| クラス | 概要 |
| - | - | 
| クラスA | ネットワーク部8ビット、ホスト部24ビット。一つのネットワークに接続できる台数は(2^24 - 2)台 = 16,777,214台 |
| クラスB | ネットワーク部16ビット、ホスト部16ビット。2^16-2台 = 65,534台 |
| クラスC ∥ ネットワーク部24ビット、ホスト部8ビット。2^8-2第 = 254台 |
| クラスD | ネットワーク部の先頭ビットが 1110 から始まる。特殊用途に限定 | 

**サブネット分割**    
ネットワーク部（サブネット含む）に1、ホスト部に0を足す(サブネットマスク)  
ネットワークをさらに分割、細分化し、効率よく使う

**CIDR(サイダー)**  
Classless Inter-Domain Routing  
サブネットをさらに進めたもの。最上位ビットから1ビット単位でネットワーク部とホスト部の境界を設定する。  

**IPv6**  
32ビットのIPv4アドレスが枯渇してきたので新しく128ビットのIPv6を策定、移行中。

## 7-05 LAN

LAN(Local Area Network)  
ある敷地内や建物内などの比較的狭い範囲内で施設したネットワーク  

**アクセス制御方式**  

* CSMA/CD方式  
  衝突したら待って再送信する
  LANに接続する端末の台数が増えるとデータの衝突が頻繁に発生するようになる

**Ethernet**  
最も普及しているLANの規格  
IEEE802.3

**無線LAN**  
IEEE802.11で標準化  
アクセス制御方式は CSMA/CA でデータの衝突を回避する

## 7-06 LAN間接続装置

**LAN間接続装置**  
LANとLAN,LANとWANを接続するときに、仕様する装置をどうする？

* リピータ  
  LANの伝送を延長するために、物理層でリンクし信号を増幅、再生して中継する。

* ブリッジ
  ブリッジは、二つのLANをデータリンク層で接続し、流れてきたパケットの宛先MACアドレスからそのパケットを送出するか否かを判断する。  

* ルータ  
  LAN同士をネットワーク層で接続し、流れてきたパケットのIPアドレスからそのパケットを送出するか否かを判断する。

* ゲートウェイ  
  トランスポート層以上のすべての層で、異なるLAN同市を接続する。

**arp(Address Resolution Protocol)**
相手のIPアドレスはわかっているが、MACアドレスがわかっていない場合に、そのMACアドレスを取得するのに使用する  
逆に、MACアドレスからIPアドレスを求めるのはRARP(ReverseARP）です
  
## 7-07 同期方式・誤り制御と伝送計算

**同期方式**  

* 調歩同期方式  
  スタートビットとストップビットで、文字単位で同期する

* キャラクタ同期方式  
  ブロックごとに先頭にSYN符号「00010110]を付与し同期をとる
  ブロック単位で制御する

* フラグ同期方式  
  常にフラグシーケンスと呼ばれる一定のビットパターン「0111110]を付与して同期をとる
  フレーム単位で同期する

**誤り制御**  

* パリティチェック  
  パリティビットを付加。1ビットの誤りを検出できる  

  * 水平パリティ  
    ビット列に対してブロック（水平）方向に0か1を付加する

  * 垂直パリティ  
    ビット列に対して文字（垂直）方向に0か1を付加する

  * 水平垂直パリティ  
    垂直パリティと水平パリティを組み合わせる

* CRC方式
  生成多項式を利用する、バースト誤りを検出できる
